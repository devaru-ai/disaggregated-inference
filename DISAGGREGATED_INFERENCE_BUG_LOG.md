# Disaggregated Inference — Bug Log & Resolutions

## Day 1: Core Infrastructure Failures
| Bug    | Error                        | Description                                         | Root Cause                                               | Solution                            | What We Did                                  | Key Commands / Fix                                |
| ------ | --------------------------------------- | --------------------------------------------------- | -------------------------------------------------------- | ----------------------------------- | -------------------------------------------- | ------------------------------------------------- |
| **1**  | `import torch` crash (NCCL mismatch)    | PyTorch fails at import due to missing NCCL symbols | PyTorch (2.6.x) incompatible with NCCL pulled by vLLM    | Pin stable versions, remove vLLM    | Downgraded PyTorch, removed conflicting deps | `pip install torch==2.4.0` + remove vllm          |
| **2**  | `Invalid device ordinal` + wrong env    | GPU indexing failure + wrong Python env             | Terminal using base env + incorrect CUDA visibility      | Use correct env, avoid GPU for test | Switched env, used CPU tensor                | `conda activate infra-platform`                   |
| **3**  | `asyncio.run()` error in Colab          | Uvicorn crashes inside Jupyter                      | Event loop already running                               | Attach to existing loop             | Replaced uvicorn.run()                       | `await server.serve()`                            |
| **4**  | `ModuleNotFoundError: src...`           | Python cannot resolve project imports               | Running script from subdir breaks path                   | Add root to PYTHONPATH              | Used proper execution method                 | `PYTHONPATH=. python ...` or `-m`                 |
| **5**  | `[Errno 98] address in use`             | Port 8080 already occupied                          | Old process still running                                | Kill process                        | Cleared port and restarted                   | `fuser -k 8080/tcp`                               |
| **6**  | Fake success (200 OK but no result)     | Ngrok returns warning page instead of API           | Missing ngrok bypass header                              | Add special header                  | Modified RPC client                          | `headers={"ngrok-skip-browser-warning": "true"}`  |
| **7**  | CUDA OOM on T4                          | Model fails to load due to VRAM limits              | 8B model exceeds 16GB constraints                        | Reduce memory footprint / quantize  | Tuned vLLM config                            | `--max-model-len 2048`, `--dtype float16`         |
| **8**  | 404 model not found / connection issues | Orchestrator cannot find correct model or endpoint  | Model name mismatch + wrong routing (localhost vs ngrok) | Use correct model ID + remote URL   | Updated payload + routing                    | `/v1/models` check + set `WORKER_URL`             |
| **9**  | `Connection refused` (ngrok)            | Tunnel exists but backend dead                      | vLLM server crashed / port conflict / IPv6 issue         | Restart env + fix binding           | Reset Colab + enforce clean startup          | `fuser -k 8000/tcp`, `ngrok http 127.0.0.1:8000`  |
| **10** | Negative KV cache (`-0.62 GiB`)         | vLLM crashes during init                            | Too low memory allocation + CUDA graphs overhead         | Increase memory + disable graphs    | Adjusted launch flags                        | `--gpu-memory-utilization 0.9`, `--enforce-eager` |
| **11** | `ERR_NGROK_3200` / 404                  | Requests reach ngrok but not backend                | No service liveness / routing guarantees                 | Add health checks + proper routing  | Diagnosed as control-plane failure           | Validate with `curl` (local + tunnel)             |
| **12** | `ALL REQUESTS FAILED`                   | Requests dropped silently                           | vLLM crashed in background (OOM)                         | Run server in foreground            | Split Colab execution                        | Run ngrok + server separately                     |
| **13** | `No supported CUDA architectures [12]`  | FlashInfer FP8 kernel compilation failure           | GPU arch mismatch (SM120 path unsupported)               | Disable FP8 / FlashInfer / JIT      | Switched to safe execution mode              | `export VLLM_USE_FLASHINFER=0`, `--enforce-eager` |
